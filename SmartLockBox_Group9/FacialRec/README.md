# FacialRec

A lightweight local face recognition application built with InsightFace ONNX
model packs and MiniFASNetV2 anti-spoofing.

## Installation

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -r requirements.txt
```

## Model Downloads

ONNX model files are not stored in this repository. Download them separately
and place the required files in the paths used by `config.json`.

```bash
mkdir -p buffalo_m buffalo_s

curl -L -o /tmp/buffalo_m.zip \
  https://github.com/deepinsight/insightface/releases/download/v0.7/buffalo_m.zip
unzip -j /tmp/buffalo_m.zip '*/det_2.5g.onnx' '*/w600k_r50.onnx' -d buffalo_m

curl -L -o /tmp/buffalo_s.zip \
  https://github.com/deepinsight/insightface/releases/download/v0.7/buffalo_s.zip
unzip -j /tmp/buffalo_s.zip '*/det_500m.onnx' '*/w600k_mbf.onnx' -d buffalo_s

curl -L -o MiniFASNetV2.onnx \
  https://github.com/yakhyo/face-anti-spoofing/releases/download/weights/MiniFASNetV2.onnx
```

Expected local model files:

```text
MiniFASNetV2.onnx
buffalo_m/det_2.5g.onnx
buffalo_m/w600k_r50.onnx
buffalo_s/det_500m.onnx
buffalo_s/w600k_mbf.onnx
```

The InsightFace model packs come from the InsightFace v0.7 release:
<https://github.com/deepinsight/insightface/releases/tag/v0.7>. The
anti-spoofing model comes from the `yakhyo/face-anti-spoofing` weights
release: <https://github.com/yakhyo/face-anti-spoofing/releases/tag/weights>.

## Model Configuration

Edit `config.json` and change `active_model_pack`:

```json
"active_model_pack": "buffalo_s"
```

Available packs:

- `buffalo_s`: smaller and faster detector and recognizer
- `buffalo_m`: larger and more accurate detector and recognizer

Each pack uses a separate database because embeddings generated by different
recognition models must not be mixed. After changing packs, enroll identities
again for the selected pack.

Anti-spoofing is enabled in `config.json`:

```json
"spoof": {
  "enabled": true,
  "model": "./MiniFASNetV2.onnx",
  "crop_scale": 2.7,
  "live_class_index": 1,
  "live_threshold": 0.6
}
```

Set `enabled` to `false` to disable spoof detection. Increase
`live_threshold` to reject more uncertain faces.

## Usage

Place enrollment photos in `face_dbs`. Each image should contain exactly one
clear, unobstructed face.

```bash
python app.py enroll David ./face_dbs/David.jpg
```

Recognize faces in an image:

```bash
python app.py recognize ./test.jpg
```

The annotated result is written to `output.jpg`.

Run real-time camera recognition:

```bash
python app.py camera
```

Press `q` in the camera window to exit.

To use another config file:

```bash
python app.py --config ./another-config.json camera
```

## Data Storage

Enrollment data contains names and normalized 512-dimensional embeddings.
Original photos are not copied into the database.

- `buffalo_s`: `data/faces_buffalo_s.npz`
- `buffalo_m`: `data/faces_buffalo_m.npz`

Biometric embeddings are sensitive data. Restrict access and obtain explicit
consent from every enrolled person.

## Limitations

MiniFASNetV2 is a single-frame RGB anti-spoof classifier. It can help reject
basic printed-photo and screen-replay attacks, but it does not guarantee
protection against high-quality displays, deepfakes, or 3D masks. High-risk
authentication should also use temporal checks or an active challenge.
