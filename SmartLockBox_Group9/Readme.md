# Smart LockBox - Secure Facial Recognition Access Control System

A cyber-physical access control system that uses facial recognition, liveness detection, MQTT communication, and digital signatures to securely authenticate users and unlock a physical lockbox.

The system captures images from a camera-equipped client device, securely transmits them to a server for analysis, and makes an authentication decision based on aggregated recognition results over a short session window.

---

## Features

* Facial recognition using InsightFace
* Anti-spoofing (liveness detection)
* MQTT communication between client and server
* Ed25519 digital signatures for secure message verification
* Multi-frame authentication aggregation
* SQLite logging system
* Image archival for every authentication attempt
* Support for both authorized and unauthorized access attempts

---

```text
+-------------------+                  +----------------------+

                                           +--------------+
```

## Authentication Workflow

1. User presses the authentication button.
2. A new authentication session begins.
3. Frames are captured for approximately 5 seconds.
4. Each frame is digitally signed and published over MQTT.
5. The server verifies the signature.
6. Facial recognition and liveness detection are performed.
7. Recognition results are aggregated over multiple frames.
8. If confidence requirements are met, the lockbox unlocks.
9. The entire session is logged.

---

## Security Features

### Digital Signature Verification

All MQTT payloads are signed using Ed25519.

Incoming messages are verified before any processing occurs.

If verification fails:

```text
Possible Spoofing Attempt!!!
```

The frame is discarded immediately.

---

## Recognition Aggregation

The system does not unlock based on a single frame.

Recognition results are aggregated across a session to improve robustness.

Unlock conditions:

* Multiple detections of the same user
* Average similarity score above a threshold

This reduces false positives from noisy detections.

## Logging System

Every authentication attempt is stored in SQLite.

### Database Schema

```sql
CREATE TABLE Logs (
    uuid TEXT PRIMARY KEY,
    session_id INTEGER,
    timestamp INTEGER,
    user TEXT,
    similarity REAL,
    lock_open BOOL,
    frame_count INTEGER,
    image_path TEXT
);
```

### Logged Information

| Field       | Description                                       |
| ----------- | ------------------------------------------------- |
| uuid        | Unique identifier for each authentication attempt |
| session_id  | Session number generated by the lockbox           |
| timestamp   | Unix timestamp                                    |
| user        | Recognized user                                   |
| similarity  | Recognition confidence                            |
| lock_open   | Lock status                                       |
| frame_count | Number of processed frames                        |
| image_path  | Saved image directory                             |

Unauthorized attempts are also logged to enable later analysis.

```bash

# For Client
cd LockBoxClient

# For Server
cd FacialRec
```

### Install Dependencies

```bash
pip install opencv-python
pip install insightface
pip install numpy
pip install pandas
pip install paho-mqtt
pip install cryptography
```

Or install using:

```bash
pip install -r requirements.txt
```

## Configuration

Update the MQTT broker settings inside `LockBoxServer.py`.

```python
self.broker
self.port
self.username
self.password
```

Place cryptographic keys inside:

```text
keys/
```

Required files:

```text
server_private_key.bin
server_public_key.bin
client_public_key.bin
```

```bash
python LockBoxServer.py
```
